# Corner Runner (가칭)

화면은 고정된 상태에서,  
2D 캐릭터가 **화면의 모서리를 벽처럼 타고 달리는** 아케이드 러너 게임입니다.

---

## 🎮 게임 개요

- **장르**: 2D 아케이드 러너
- **플랫폼**: PC (Unity)
- **플레이 방식**: 자동 이동 + 타이밍 기반 회피
- **레퍼런스**: 컴투스 *놈*

---

## 🧠 핵심 콘셉트

- 캐릭터는 화면 안을 자유롭게 이동하지 않습니다.
- **화면 테두리(사각형 경로)** 만을 따라 이동합니다.
- 이동 방향은 고정되어 있으며, 플레이어는 방향을 조작하지 않습니다.
- 화면은 **회전하지 않습니다.**
- 캐릭터가 벽을 바닥처럼 타고 달리는 이질적인 시각적 경험을 제공합니다.

> “위는 항상 위, 아래는 항상 아래  
> 그런데 캐릭터는 벽을 달린다”

---

## 📐 이동 규칙

캐릭터는 **반시계 방향 루프**로 사각형 테두리를 순환합니다.

| 현재 위치(벽) | 캐릭터 이동 방향 |
|-------------|----------------|
| 하단 벽 | 오른쪽 → |
| 오른쪽 벽 | 위 ↑ |
| 상단 벽 | 왼쪽 ← |
| 왼쪽 벽 | 아래 ↓ |

- 코너에 도달하면 자동으로 다음 벽으로 전환됩니다.
- 이동은 완전 자동이며, 중력은 사용하지 않습니다.

---

## 🎥 카메라 & 화면 연출

- **고정 카메라**
- 화면 회전 없음
- 하나의 화면 안에 사각형 테두리 전체가 보이도록 구성
- 캐릭터가 어느 벽을 달리든 화면 기준 방향은 유지됨

---

## 🕹 플레이어 조작

- 이동: 자동
- 입력: 최소 1버튼

### 액션 (예시)
- 점프 / 회피

#### 점프 규칙
- 점프 방향은 **현재 벽의 바깥쪽 방향**으로 자동 결정
- 플레이어는 타이밍만 신경 쓰면 됨

---

## 🚧 장애물 시스템

- 장애물은 사각형 경로 위에만 등장
- 캐릭터와 동일한 **경로 진행도(progress)** 좌표계 사용
- 물리 충돌이 아닌, 수식 기반 판정으로 처리

---

## ⚙️ 기술적 설계 요약

- 사각형 둘레를 하나의 **1차원 경로(progress)** 로 관리
- `progress` 값에 따라 위치 계산
- Rigidbody 및 중력 없이 구현
- 캐릭터, 장애물, 이벤트 모두 동일한 좌표계 사용

---

## 🔥 게임의 특징

- 화면이 회전하지 않는 벽 타기 러너
- 방향 조작 제거로 인한 높은 접근성
- 단순하지만 명확한 기하학적 구조
- 빠른 템포의 반복 플레이에 최적화된 설계
